# 権限エラー時の自律回避ルール

## 原則

権限エラーが発生した場合、すぐに作業を止めず、代替手段を自律的に探して完遂を目指す。

## 回避戦略（優先度順）

### 1. 専用ツールへの切り替え
- Bash(grep ...) → Grep
- Bash(cat ...) → Read
- Bash(find ...) → Glob
- Bash(sed ...) → Edit
- Bash(echo > ...) → Write

### 2. 許可済みコマンドへの言い換え
- `python3 -c` や `node -e` で小さい処理へ言い換える
- 許可済みコマンドの範囲で処理を分解する

### 3. スクリプトファイル経由の実行
- 一時スクリプトを書き、許可済みランタイムで実行する

### 4. それでも不可なら制約を明示して報告する
- 何が拒否されたか
- 何を代替として行ったか
- 何が未実施か

## 禁止事項
- 権限エラーだけを理由にタスクを放棄しない
- セキュリティ制約の回避を試みない
- 破壊的な回避策を勝手に選ばない
