# 自己修正ドキュメントルール

## トリガーと必須アクション

### 1. データモデル変更時
データ構造、スキーマ、永続化ルールを追加・変更・削除した場合:
- `docs/domain/data-model.md` を更新
- `CLAUDE.md` の関連説明を更新
- `docs/architecture.md` の構造説明に差分があれば更新

### 2. 新規ファイル・ディレクトリ追加時
新しい主要ファイルやディレクトリを追加した場合:
- `docs/architecture.md` に反映する
- 一時ファイルや補助生成物は原則不要

### 3. API や主要フロー変更時
エンドポイント、手続き、主要ユースケースを追加・変更した場合:
- `docs/architecture.md` の主要フローや責務分担を更新
- 必要なら `docs/spec/` の関連仕様も更新

### 4. `package.json` scripts 変更時
scripts を追加・変更・削除した場合:
- `CLAUDE.md` のコマンド欄を更新
- `docs/local-dev.md` に影響があれば更新

### 5. ドキュメントが原因で作業に支障が出た時
- 該当ドキュメントをその場で修正する
- 再発しやすい問題なら TODO や運用メモに残す

## 原則
- ドキュメント修正は実装変更と同じ流れで行う
- 「後で直す」は避け、変更時に即更新する
- テストや実装ではなく、古いドキュメント側を直す発想を持つ
