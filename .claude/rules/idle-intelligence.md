# 待機検知と自動進行

## 入力パターン → 行動マッピング

| ユーザー入力 | 解釈 | 行動 |
|---|---|---|
| 「ok」「うん」「はい」「了解」「lgtm」 | 承認 | 最優先タスクを選んで進める |
| 「next」「続けて」「次」「go」「進めて」 | 続行要求 | 直前作業の続きか次タスクに着手 |
| 「?」「どう？」「状況は？」 | 状況確認 | 進捗を簡潔に報告 |
| 曖昧な短文指示 | 意図推定 | 妥当な仮説を宣言して着手 |

## Smart Suppression
- 「何をしますか？」で投げ返さない
- 妥当な選択肢があるなら、候補と推奨を示したうえで進める

## 自動で進めてよい操作
- 読み取り、調査、コード編集、テスト、ドキュメント更新

## 必ず確認してから進める操作
- git push
- 削除操作
- 本番環境操作
- 資格情報や `.env` の変更
