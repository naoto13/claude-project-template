# 仕様駆動実装ワークフロー

## フェーズ 1: 事前調査

1. `docs/spec/` の対象仕様を読む
2. `docs/architecture.md` で変更レイヤーと依存方向を確認する
3. 既存実装を調査し、変更ファイルを洗い出す
4. 実装計画を短く共有する

## フェーズ 2: TDD

### Red
- Acceptance Criteria をテストへ落とす
- 失敗を確認する

### Green
- 最小限の実装で通す
- 既存パターンに合わせる
- プロジェクトのログ規約と例外処理規約に従う

### Refactor
- 可読性、重複、責務分離を改善する

## フェーズ 3: 検証

利用可能なコマンドを実行する。

```bash
pnpm test
pnpm check
pnpm build
```

存在しない場合は、そのプロジェクトの等価コマンドで代替する。

## フェーズ 4: 完了報告

- 変更ファイル
- 追加または更新したテスト
- Acceptance Criteria の充足状況
- 実行した検証コマンド
