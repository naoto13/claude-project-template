# 仕様駆動実装ワークフロー

このワークフローの目的は、確定済みの仕様書に基づいて、TDD 原則で実装と検証を完了させることです。

## フェーズ 1: 事前調査

1. **仕様書を熟読**: `docs/spec/feature-*.md` の Acceptance Criteria を把握
2. **アーキテクチャを確認**: `docs/architecture.md` で変更レイヤーと依存方向を確認
3. **既存実装を調査**: 対象ファイルを Read / Grep で確認し、影響範囲を把握
4. **実装計画を宣言**: どのファイルに何を変更するかを短く共有

## フェーズ 2: テスト駆動開発

### Red
- Acceptance Criteria をテストケースに変換する
- テストが失敗することを確認する

### Green
- 最小限の実装でテストを通す
- 依存方向、ログ規約、例外処理規約、既存パターンを守る

### Refactor
- 可読性、重複、責務分離を改善する

## フェーズ 3: 検証

以下を基本として、存在するコマンドを実行する。

```bash
pnpm test
pnpm check
pnpm build
```

- コマンドが存在しない場合は、等価コマンドで代替する
- テスト、型チェック、ビルドのうち何を実行したかを完了報告に含める

## フェーズ 4: TODO更新と完了報告

### TODO更新
検証がパスしたら、`docs/todo.md` を確認し:
- 実装した機能に対応する `[ ]` 項目を `[x]` に更新
- 該当項目がなければ、適切なセクションに完了項目として追記

### 完了報告
- 変更ファイル一覧と差分サマリー
- テスト結果
- Acceptance Criteria の充足状況
- 実行した検証コマンド
