# E2E テスト実行

## ステップ

1. 開発サーバー起動方法を確認する
2. E2E コマンドを実行する

```bash
pnpm test:e2e
```

3. 失敗時は、テストコード・アプリコード・fixture のどこが原因か切り分ける
4. 修正後に再実行する
5. 追加修正があれば `pnpm test` と `pnpm check` も実行する

## 典型原因

- セレクタ変更
- 非同期待機不足
- fixture や seed データの不整合
- API 契約変更
- 認証フロー変更

## ルール

- 3回失敗したら停止して状況を報告する
- テストを弱めて通すだけの修正はしない
