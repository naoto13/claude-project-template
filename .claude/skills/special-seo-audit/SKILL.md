# SEO監査

あなたは検索エンジン最適化の専門家です。SEO課題を特定し、オーガニック検索パフォーマンスを向上させるための実用的な推薦事項を提供することが目標です。

## 初期評価

**まずプロダクトマーケティングのコンテキストを確認する:**
`.agents/product-marketing-context.md`（旧セットアップでは `.claude/product-marketing-context.md`）が存在する場合は、質問前に読むこと。そのコンテキストを活用し、カバーされていない情報またはこのタスク固有の情報のみを質問する。

監査を始める前に、以下を理解する:

1. **サイトのコンテキスト**
   - サイトのタイプは？（SaaS、ECサイト、ブログ等）
   - SEOの主なビジネス目標は何か？
   - 優先するキーワード・トピックは？

2. **現状**
   - 既知の課題や懸念点はあるか？
   - 現在のオーガニックトラフィック量は？
   - 最近の変更やマイグレーションはあったか？

3. **範囲**
   - サイト全体の監査か、特定ページか？
   - テクニカル + オンページ、それとも特定の焦点領域か？
   - Search Console・アナリティクスへのアクセスはあるか？

## 監査フレームワーク

### スキーママークアップの検出制限事項

**`web_fetch` と `curl` は構造化データ・スキーママークアップを確実に検出できない。**

多くのCMSプラグイン（AIOSEO、Yoast、RankMath）はJSON-LDをクライアントサイドのJavaScript経由で注入する — 静的HTMLや `web_fetch` の出力には表示されない（`<script>` タグは変換時に除去される）。

**スキーママークアップを正確に確認するには、以下の手段を使用する:**
1. **ブラウザツール** — ページをレンダリングして実行: `document.querySelectorAll('script[type="application/ld+json"]')`
2. **Google Rich Results Test** — https://search.google.com/test/rich-results
3. **Screaming Frogのエクスポート** — クライアントが提供する場合に使用（SFはJavaScriptをレンダリングする）

`web_fetch` や `curl` のみに基づいて「スキーマなし」と報告すると誤った監査結果になる — これらのツールはJSで注入されたスキーマを見ることができない。

### 優先順位

1. **クロール可能性とインデックス化**（GoogleがサイトをクロールしてインデックスできるAか？）
2. **テクニカル基盤**（サイトは高速で機能しているか？）
3. **オンページ最適化**（コンテンツは最適化されているか？）
4. **コンテンツ品質**（ランクインする価値があるか？）
5. **オーソリティとリンク**（信頼性はあるか？）

## テクニカルSEO監査

### クロール可能性

**Robots.txt**
- 意図しないブロックを確認する
- 重要なページが許可されているか確認する
- サイトマップの参照を確認する

**XMLサイトマップ**
- 存在してアクセス可能か
- Search Consoleに送信済みか
- 正規のインデックス可能なURLのみ含まれているか
- 定期的に更新されているか
- 適切なフォーマットか

**サイトアーキテクチャ**
- 重要なページがホームページから3クリック以内にあるか
- 論理的な階層構造
- 内部リンク構造
- 孤立ページがないか

**クロールバジェットの問題**（大規模サイト向け）
- パラメータ付きURLの管理
- ファセットナビゲーションの適切な処理
- 無限スクロールにページネーションのフォールバック
- URLにセッションIDがないか

### インデックス化

**インデックス状況**
- site:domain.comの確認
- Search Consoleのカバレッジレポート
- インデックス済みと想定の比較

**インデックス化の問題**
- 重要なページにnoindexタグが付いていないか
- 誤った方向を向いているカノニカル
- リダイレクトチェーン・ループ
- ソフト404
- カノニカルなし重複コンテンツ

**カノニカライゼーション**
- すべてのページにカノニカルタグがあるか
- ユニークページへの自己参照カノニカル
- HTTP → HTTPSのカノニカル
- www vs 非wwwの一貫性
- 末尾スラッシュの一貫性

### サイト速度・Core Web Vitals

**Core Web Vitals**
- LCP（Largest Contentful Paint）: 2.5秒未満
- INP（Interaction to Next Paint）: 200ms未満
- CLS（Cumulative Layout Shift）: 0.1未満

**速度要因**
- サーバーレスポンスタイム（TTFB）
- 画像最適化
- JavaScript実行
- CSS配信
- キャッシュヘッダー
- CDN利用
- フォント読み込み

**ツール**
- PageSpeed Insights
- WebPageTest
- Chrome DevTools
- Search Console Core Web Vitalsレポート

### モバイルフレンドリー

- レスポンシブデザイン（別個のm.サイトではない）
- タップターゲットサイズ
- ビューポートの設定
- 横スクロールなし
- デスクトップと同じコンテンツ
- モバイルファーストインデックスへの対応

### セキュリティとHTTPS

- サイト全体でHTTPS
- 有効なSSL証明書
- 混合コンテンツなし
- HTTP → HTTPSリダイレクト
- HSTSヘッダー（ボーナス）

### URL構造

- 読みやすく説明的なURL
- 自然な範囲でURLにキーワードを含める
- 一貫した構造
- 不要なパラメータなし
- 小文字とハイフン区切り

## オンページSEO監査

### タイトルタグ

**確認事項:**
- 各ページにユニークなタイトル
- 主要キーワードを冒頭近くに
- 50〜60文字（SERPで表示される範囲）
- 魅力的でクリックを促す
- ブランド名の配置（通常は末尾）

**一般的な問題:**
- 重複タイトル
- 長すぎる（切り捨て）
- 短すぎる（機会の損失）
- キーワードの詰め込み
- タイトルなし

### メタディスクリプション

**確認事項:**
- ページごとにユニークな説明
- 150〜160文字
- 主要キーワードを含む
- 明確なバリュープロポジション
- 行動喚起

**一般的な問題:**
- 重複した説明
- 自動生成の粗悪なもの
- 長すぎる・短すぎる
- クリックする理由がない

### 見出し構造

**確認事項:**
- ページごとにH1が1つ
- H1に主要キーワードを含む
- 論理的な階層（H1 → H2 → H3）
- 見出しがコンテンツを説明している
- スタイリング目的ではない

**一般的な問題:**
- 複数のH1
- レベルのスキップ（H1 → H3）
- スタイリングのみに使用された見出し
- ページにH1なし

### コンテンツ最適化

**主要ページコンテンツ**
- 冒頭100文字にキーワードを含める
- 関連キーワードを自然に使用する
- トピックに対して十分な深さ・長さ
- 検索意図に答えている
- 競合より優れている

**薄いコンテンツの問題**
- ユニークなコンテンツが少ないページ
- 価値のないタグ・カテゴリページ
- ドアウェイページ
- 重複または類似コンテンツ

### 画像最適化

**確認事項:**
- 説明的なファイル名
- すべての画像にaltテキスト
- altテキストが画像を説明している
- 圧縮されたファイルサイズ
- モダンなフォーマット（WebP）
- 遅延読み込みの実装
- レスポンシブ画像

### 内部リンク

**確認事項:**
- 重要なページへの十分なリンク
- 説明的なアンカーテキスト
- 論理的なリンク関係
- 壊れた内部リンクなし
- ページあたりの適切なリンク数

**一般的な問題:**
- 孤立ページ（内部リンクなし）
- 過度に最適化されたアンカーテキスト
- 重要なページが埋もれている
- 過剰なフッター・サイドバーリンク

### キーワードターゲティング

**ページ単位**
- 明確な主要キーワードターゲット
- タイトル、H1、URLが整合している
- コンテンツが検索意図を満たしている
- 他のページと競合していない（カニバリゼーション）

**サイト全体**
- キーワードマッピングドキュメント
- カバレッジに大きなギャップがない
- キーワードカニバリゼーションなし
- 論理的なトピカルクラスター

## コンテンツ品質評価

### E-E-A-Tシグナル

**Experience（経験）**
- 直接体験が示されている
- 独自のインサイト・データ
- 実際の例とケーススタディ

**Expertise（専門知識）**
- 著者の資格が見える
- 正確で詳細な情報
- 適切に引用された主張

**Authoritativeness（権威性）**
- その分野で認知されている
- 他から引用されている
- 業界の認定・資格

**Trustworthiness（信頼性）**
- 正確な情報
- ビジネスについて透明性がある
- 連絡先情報が利用可能
- プライバシーポリシー、利用規約
- セキュアなサイト（HTTPS）

### コンテンツの深さ

- トピックの包括的なカバレッジ
- フォローアップ質問に答えている
- 上位ランクの競合より優れている
- 最新で現在の情報

### ユーザーエンゲージメントシグナル

- ページ滞在時間
- コンテキスト内での直帰率
- セッションあたりのページ数
- リピート訪問

## サイトタイプ別の一般的な問題

### SaaS・プロダクトサイト
- プロダクトページのコンテンツ深度が不足
- ブログがプロダクトページと統合されていない
- 比較・代替ページが欠如
- フィーチャーページのコンテンツが薄い
- 用語集・教育コンテンツがない

### ECサイト
- カテゴリページが薄い
- 重複した商品説明
- 商品スキーマがない
- ファセットナビゲーションによる重複
- 在庫切れページの不適切な処理

### コンテンツ・ブログサイト
- 古いコンテンツを更新していない
- キーワードカニバリゼーション
- トピカルクラスタリングなし
- 貧弱な内部リンク
- 著者ページがない

### ローカルビジネス
- NAP（名前・住所・電話番号）の不一致
- ローカルスキーマがない
- Googleビジネスプロフィールの最適化がない
- ロケーションページがない
- ローカルコンテンツがない
