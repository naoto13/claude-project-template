## Dropbox（クラシック）

**プログラム:** 500MBのストレージを贈り、500MBのストレージを受け取る

**効果的だった理由:**
- 報酬がプロダクト価値に直接紐づいている
- 低い摩擦（メールだけ）
- 両者が平等にメリットを受ける
- 進捗トラッキングでゲーミフィケーション

---

## Uber/Lyft

**プログラム:** 10ドルのライドクレジットを贈り、相手が乗車すると10ドルを受け取る

**効果的だった理由:**
- 即時で明確な価値
- 両側インセンティブ
- シェアしやすい（コード・リンク）
- 自然なモーメントにトリガー

---

## Morning Brew

**プログラム:** サブスクライバーの紹介に対する段階報酬
- 3人紹介: ニュースレターステッカー
- 5人紹介: Tシャツ
- 10人紹介: マグカップ
- 25人紹介: パーカー

**効果的だった理由:**
- ゲーミフィケーションが継続的なエンゲージメントを促進
- 物理的な報酬はシェアしやすい（さらなる紹介につながる）
- サブスクライバー価値に対してコストが低い
- ステータス・アイデンティティを構築した

---

## Notion

**プログラム:** 紹介ごとに10ドルのクレジット（教育向け）

**効果的だった理由:**
- シェアしやすいオーディエンス（学生）にターゲット
- プロダクトがチームで自然に広がる
- クレジットがユーザーのエンゲージメントを維持

## インセンティブタイプの比較

| タイプ | メリット | デメリット | 最適な用途 |
|------|------|------|----------|
| 現金・クレジット | 普遍的に価値がある | 取引的な印象 | マーケットプレイス、フィンテック |
| プロダクトクレジット | 使用を促進 | 使わなければ価値なし | SaaS、サブスクリプション |
| 無料月数 | 明確な価値 | 無料特典目当てを引き寄せる可能性 | サブスクリプション製品 |
| フィーチャーアンロック | コストが低い | ゲート機能がある場合のみ有効 | フリーミアム製品 |
| グッズ・ギフト | 記憶に残る、シェアされやすい | 物流の複雑さ | ブランド重視の企業 |
| チャリティ寄付 | 良い気持ちになれる | 個人的な動機が低い | ミッション志向ブランド |

## インセンティブサイジングフレームワーク

**最大インセンティブの計算:**
```
最大紹介報酬 = (顧客LTV × 粗利率) - 目標CAC
```

**例:**
- LTV: 1,200ドル
- 粗利率: 70%
- 目標CAC: 200ドル
- 最大報酬: (1,200 × 0.70) - 200 = 640ドル

**一般的な紹介報酬:**
- B2C: 10〜50ドルまたは初回購入の10〜25%
- B2B SaaS: 50〜500ドルまたは1〜3ヶ月無料
- エンタープライズ: より高額で、カスタムが多い

## バイラル係数・指標

### 主要指標

**バイラル係数（K係数）:**
```
K = 招待数 × コンバージョン率

K > 1 = バイラル成長（各ユーザーが1人以上の新規ユーザーを連れてくる）
K < 1 = 増幅成長（紹介が他の獲得を補完する）
```

**例:**
- 平均顧客が3件の招待を送る
- 招待の15%がコンバートする
- K = 3 × 0.15 = 0.45

**紹介率:**
```
紹介率 = (紹介した顧客数) / (顧客総数)
```

ベンチマーク:
- 良好: 顧客の10〜25%が紹介
- 優秀: 25〜50%
- 卓越: 50%以上

**紹介者あたりの紹介数:**

ベンチマーク:
- 平均: 紹介者あたり1〜2件
- 良好: 2〜5件
- 卓越: 5件以上

### 紹介プログラムROIの計算

```
紹介プログラムROI = (紹介顧客からの収益 - プログラムコスト) / プログラムコスト

プログラムコスト = 支払われた報酬 + ツールコスト + 管理工数
```

**別途追跡する:**
- 紹介顧客あたりのコスト（紹介経由のCAC）
- 紹介顧客のLTV（多くの場合、平均より高い）
- 紹介報酬の回収期間
