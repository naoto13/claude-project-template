# ワンページャーテンプレート

異なるユースケースのワンページャーテンプレート。レイアウトガイダンスとコピープロンプト付き。

## プロダクト概要ワンページャー

デフォルトのワンページャー。あなたについて何も知らない人に製品を紹介する。

### 構成

```
[ロゴ]                                              [タグライン]

ヘッドライン: 何をする会社で、誰向けかを1文で説明。

問題
購買者が直面しているペインを2〜3文で説明。

ソリューション
製品がそれをどのように解決するかを2〜3文で説明。

なぜ[製品名]か
• 差別化要因1 — メリットを1文で説明
• 差別化要因2 — メリットを1文で説明
• 差別化要因3 — メリットを1文で説明

証拠
「具体的な結果を含む顧客の引用。」— 名前、役職、会社
[オプション: 2〜3つの指標コールアウト: 「X%改善」「Y時間節約」]

[CTA ボタン・リンク]                    [連絡先: name@company.com]
```

### コピープロンプト

- ヘッドライン: 「1文で何をするか。それを聞いた人が『もっと教えて』と言うものは？」
- 問題: 「購買者はあなたを見つける前に何に苦しんでいるか？」
- 差別化要因: 「3つしか伝えられないとしたら、何があなたを選ばせるか？」

## ユースケース固有のワンページャー

特定のワークフロー、バーティカル、または問題に合わせたもの。プロダクト概要より具体的。

### 構成

```
[ロゴ]                                    [ユースケース: 例「セールスチーム向け」]

ヘッドライン: [製品]が[ペルソナ]の[成果達成]をどのように支援するか。

課題
[ペルソナ]が[タスク]を必要とする時、[具体的なペイン]に直面する。
これは[結果]につながる: [無駄な時間 / 失ったお金 / リスク]。

仕組み
1. [ステップ1] — 何が起きて、なぜ重要か
2. [ステップ2] — 何が起きて、なぜ重要か
3. [ステップ3] — 何が起きて、なぜ重要か

結果
• [指標1]: 前 → 後
• [指標2]: 前 → 後
• [指標3]: 前 → 後

顧客スポットライト
「このユースケースについての引用。」 — 名前、役職、会社

[CTA: 「実際に見る」または「パイロットを開始」]       [連絡先]
```

### 使用すべきタイミング

- 異なる購買者ペルソナには異なるワンページャーが必要
- 業界固有のバージョン（ヘルスケア、フィンテック、EC）
- ユースケースバージョン（レポーティング、オンボーディング、セキュリティ）

## ミーティング後リーブビハインド

既に行われた会話を強化するために設計。話し合った内容をサマリーし、次のステップを提案する。

### 構成

```
[ロゴ]                                            [ミーティング日付]

ミーティングリキャップ: [会社名]

話し合った内容
• [彼らが述べたペインポイント1]
• [彼らが述べたペインポイント2]
• [達成しようとしている目標]

[製品]がどのように役立つか
• [ペイン1へのソリューション] — [特定の機能またはワークフロー]
• [ペイン2へのソリューション] — [特定の機能またはワークフロー]
• [目標達成のサポート方法]

関連する証拠
「類似した顧客からの引用。」 — 名前、役職、会社
[類似した顧客からの1〜2つの指標]

提案する次のステップ
1. [日付付きの次のステップ]
2. [フォローアップアクション]
3. [意思決定タイムライン]

[あなたの名前]  |  [役職]  |  [メール]  |  [電話]
```

### ヒント

- ミーティングから24時間以内に送る
- 彼らが言った具体的なことを参照する（聞いていたことを示す）
- 次のステップを具体的で時間を区切って提示する
- これはチャンピオンが上司に転送するアセット

## チャンピオンイネーブルメントワンページャー

内部チャンピオンがチームとリーダーシップに共有するために特別に設計されたもの。チャンピオンが賢く見えるように書かれている。

### 構成

```
[ロゴ]

なぜ[製品名]を評価しているのか

状況
[内部課題についての2〜3文。チャンピオンがチームに
説明しているように書く。「私たち」と「私たちの」の言葉を使う。]

[製品]がすること
[1〜2文。平易な言葉で、専門用語なし。]

なぜこのソリューションか
• [理由1] — 私たちの具体的な問題をどのように解決するか
• [理由2] — 現在の行い方とどのように比較するか
• [理由3] — 評価した代替案とどのように比較するか

期待されるインパクト
• [指標]: 現状 → 期待される状態
• [指標]: 現状 → 期待される状態
• 価値実現までの時間: [X週間]以内にライブ

他の利用企業
[業界の認知度が高い2〜3社名]
「関連する顧客の引用。」 — 名前、役職、会社

次のステップ
• [次にすること]
• [チームに必要なもの]
• [意思決定タイムライン]

質問は[チャンピオン名]または[メールのあなたの名前]まで。
```

### なぜこれが機能するか

- あなたのものではなく、チャンピオンの声で書かれている
- 上司が聞く質問に答えている
- 彼らが尊重する企業からの同業者の証拠を含む
- 社内のモメンタムを促進するための明確なお願いとタイムライン

## レイアウトガイダンス

### 視覚的階層

1. **ヘッドライン** — 最大のテキスト、ページの上部、価値を即座に伝える
2. **セクションヘッダー** — 太字、明確、スキャン可能なアンカーとして機能
3. **本文テキスト** — 短い文、段落よりも箇条書きが好ましい
4. **証拠要素** — 指標と引用は視覚的に目立つ（大きいフォント、色、またはコールアウトボックス）
5. **CTA** — 目立つ配置、ページの下部または右下

### 余白

- マージン: すべての辺に少なくとも0.75インチ
- セクション間のスペース: 視覚的に分離するのに十分（詰め込まない）
- 混雑している感じがすれば、コンテンツを削減する。フォントを9pt以下に縮小しない。

### フォントサイズ

| 要素 | 推奨サイズ |
|---------|---------------|
| ヘッドライン | 18〜24pt |
| セクションヘッダー | 12〜14pt太字 |
| 本文テキスト | 10〜11pt |
| 細字 / フッター | 8〜9pt |

### カラー

- ヘッダーとアクセントにブランドカラーを使用する
- 本文テキストは白地に濃い（黒または黒に近い）色を保つ
- アクセントカラーは視覚的一貫性のために1〜2色に制限する
- 指標とCTAに注目を引くためにカラーを使用する

### ファイルフォーマット

- **PDF**: メールの添付ファイルとリーブビハインド用
- **Google Slides / PowerPoint**: 担当者がカスタマイズできる編集可能バージョン
- 常に両方を用意する — 担当者はカスタマイズし、見込み客はクリーンなPDFを求める
