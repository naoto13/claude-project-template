# 競合ページのセクションテンプレート

競合比較ページの各セクションにすぐ使えるテンプレート集。

## 目次
- TL;DR サマリー
- 段落比較（テーブルだけに頼らない）
- 機能比較セクション
- 価格比較セクション
- サービス・サポート比較
- 向いているユーザーセクション
- 移行セクション
- ソーシャルプルーフセクション
- 比較テーブルのベストプラクティス（チェックマークを超えた表現、カテゴリ別整理、評価の活用）

## TL;DR サマリー

スキャンするユーザー向けに、すべてのページを簡潔なサマリーで始める：

```markdown
**TL;DR**：[競合]は[強み]に優れているが、[弱み]に課題がある。
[自社製品]は[フォーカス]向けに構築されており、[主な差別化要因]を提供する。
[特定のユースケース]なら[競合]を選ぶ。[特定のユースケース]なら[自社]を選ぶ。
```

## 段落比較（テーブルだけに頼らない）

主要な軸ごとに段落を書く：

```markdown
## 機能

[競合]は[機能のアプローチ説明]を提供する。
特に[具体的な強み]が優れており、[ユースケース]に適している。
ただし、[制限]は[ユーザータイプ]にとって課題になる場合がある。

[自社製品]は[自社のアプローチ]という異なる方針を取っている。
これにより[メリット]が得られるが、[正直なトレードオフ]もある。
[特定のニーズ]を持つチームはこちらをより効果的と感じることが多い。
```

## 機能比較セクション

チェックマークを超えた表現にする：

```markdown
## 機能比較

### [機能カテゴリ]

**[競合]**：[対応方法を2〜3文で説明]
- 強み：[具体的に]
- 制限：[具体的に]

**[自社製品]**：[対応方法を2〜3文で説明]
- 強み：[具体的に]
- 制限：[具体的に]

**結論**：[このシナリオ]なら[競合]を選ぶ。[このシナリオ]なら[自社]を選ぶ。
```

## 価格比較セクション

```markdown
## 価格

| | [競合] | [自社製品] |
|---|---|---|
| 無料プラン | [詳細] | [詳細] |
| 開始価格 | $X/ユーザー/月 | $X/ユーザー/月 |
| ビジネスプラン | $X/ユーザー/月 | $X/ユーザー/月 |
| エンタープライズ | 要問い合わせ | 要問い合わせ |

**含まれる内容**：[競合]の$Xプランには[機能]が含まれ、
[自社製品]の$Xプランには[機能]が含まれる。

**コスト総額の考慮事項**：席単価以外に[隠れたコスト・
アドオン・導入費用]も考慮する必要がある。[競合]は[X]に追加費用がかかるが、
[自社製品]は[Y]を基本料金に含んでいる。

**費用対効果の比較**：10人チームの場合、[競合]は年間約$X、
[自社製品]は年間約$Yとなり、[主な違い]がある。
```

## サービス・サポート比較

```markdown
## サービス・サポート

| | [競合] | [自社製品] |
|---|---|---|
| ドキュメント | [品質評価] | [品質評価] |
| 応答時間 | [SLA（判明している場合）] | [自社のSLA] |
| サポートチャネル | [一覧] | [一覧] |
| オンボーディング | [提供内容] | [提供内容] |
| CSM担当 | [対象プラン] | [対象プラン] |

**サポート品質**：[G2/Capterra レビュー・調査]によると、
[競合]のサポートは[評価]と評されている。よくある声として
[引用やテーマ]がある。

[自社製品]は[自社のサポートアプローチ]を提供している。[応答時間・
専任CSM・導入支援など具体的な差別化ポイント]がある。
```


## ソーシャルプルーフセクション

乗り換えユーザーにフォーカスする：

```markdown

> "[乗り換えた理由と成果についての具体的なコメント]"
> — [氏名]、[役職] at [会社名]

> "[別のコメント]"
> — [氏名]、[役職] at [会社名]

### 乗り換え後の成果
- [会社名]は[具体的な成果]を達成した
- [会社名]は[指標]を[数値]削減した
```

## 比較テーブルのベストプラクティス

### チェックマークを超えた表現

以下の代わりに：
| 機能 | 自社 | 競合 |
|---------|-----|-----------|
| 機能A | ✓ | ✓ |
| 機能B | ✓ | ✗ |

こうする：
| 機能 | 自社 | 競合 |
|---------|-----|-----------|
| 機能A | [詳細]を含む完全対応 | 基本対応のみ、[制限あり] |
| 機能B | [具体的な機能] | 非対応 |

### カテゴリ別に整理する

機能を意味のあるカテゴリにグループ化する：
- コア機能
- コラボレーション
- インテグレーション
- セキュリティ・コンプライアンス
- サポート・サービス

### 有用な場合は評価を含める

| カテゴリ | 自社 | 競合 | 補足 |
|----------|-----|-----------|-------|
| 使いやすさ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | [簡単なメモ] |
| 機能の深さ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | [簡単なメモ] |
