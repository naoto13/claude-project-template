# 競合ページのコンテンツ設計

スケーラブルな比較ページのために競合データを構造化・管理する方法。

## 目次
- 競合データの一元管理
- 競合データテンプレート
- 自社製品データ
- ページ生成
- インデックスページ構成（代替インデックス、比較インデックス、インデックスページのベストプラクティス）
- フッターナビゲーション

## 競合データの一元管理

各競合につき唯一の情報源を作成する：

```
competitor_data/
├── notion.md
├── airtable.md
├── monday.md
└── ...
```

## 競合データテンプレート

競合ごとに以下を記録する：

```yaml
name: Notion
website: notion.so
tagline: "The all-in-one workspace"
founded: 2016
headquarters: San Francisco

# ポジショニング
primary_use_case: "ドキュメント + 軽量データベース"
target_audience: "柔軟なワークスペースを求めるチーム"
market_position: "プレミアム・機能豊富"

# 価格
pricing_model: per-seat
free_tier: true
free_tier_limits: "ブロック数制限あり、1ユーザー"
starter_price: $8/user/month
business_price: $15/user/month
enterprise: custom

# 機能（1〜5で評価または説明）
features:
  documents: 5
  databases: 4
  project_management: 3
  collaboration: 4
  integrations: 3
  mobile_app: 3
  offline_mode: 2
  api: 4

# 強み（正直に）
strengths:
  - 非常に柔軟でカスタマイズ性が高い
  - 美しくモダンなインターフェース
  - 豊富なテンプレートエコシステム
  - アクティブなコミュニティ

# 弱み（公平に）
weaknesses:
  - 大規模データベースでは動作が遅くなる場合がある
  - 高度な機能には学習コストがかかる
  - 専用ツールと比べて自動化機能が限定的
  - オフラインモードが制限されている

# 向いているユーザー
best_for:
  - オールインワンワークスペースを求めるチーム
  - コンテンツ重視のワークフロー
  - ドキュメントファーストのチーム
  - スタートアップや小規模チーム

# 向いていないユーザー
not_ideal_for:
  - 複雑なプロジェクト管理が必要なチーム
  - 大規模データベース（数千行）を扱う場合
  - 堅牢なオフライン機能が必要なチーム
  - 厳格なコンプライアンスが必要なエンタープライズ

# よくある不満（レビューより）
common_complaints:
  - "コンテンツが増えると動作が遅くなる"
  - "ワークスペースが大きくなると情報を見つけにくい"
  - "モバイルアプリが使いにくい"

# 移行メモ
migration_from:
  difficulty: medium
  data_export: "Markdown, CSV, HTML"
  what_transfers: "ページ、データベース"
  what_doesnt: "自動化設定、インテグレーション設定"
  time_estimate: "小規模チームなら1〜3日"
```

## ページ生成

各ページは一元管理されたデータから情報を取得する：

- **[競合]の代替ページ**：競合データ + 自社データを使用
- **[競合]の代替一覧ページ**：競合データ + 自社データ + 他の代替ツールを使用
- **自社 vs [競合]ページ**：自社データ + 競合データを使用
- **[A] vs [B]ページ**：両競合データ + 自社データを使用

**メリット**：
- 競合の価格を1回更新するだけで全ページに反映される
- 新しい機能比較を1回追加するだけで全ページに表示される
- 全ページで一貫した正確性を保てる
- 大規模運用でもメンテナンスが容易

## インデックスページ構成

### 代替インデックス

**URL**：`/alternatives` または `/alternatives/index`

**目的**：すべての「[競合]の代替」ページを一覧表示する

**ページ構成**：
1. 見出し：「[自社製品]という選択肢」
2. 乗り換える理由の簡単な説明
3. 全代替ページの一覧（以下を含む）：
   - 競合名/ロゴ
   - その競合との主な差別化ポイントを1行で
   - 詳細比較へのリンク
4. 乗り換えのよくある理由（集約）
5. CTA

**例**：
```markdown
## [自社製品]への乗り換えを検討中ですか？

[自社製品]が検討中のツールとどう違うかをご確認ください：

- **[Notion の代替](/alternatives/notion)** — [X]が必要なチームに最適
- **[Airtable の代替](/alternatives/airtable)** — [Y]が必要なチームに最適
- **[Monday の代替](/alternatives/monday)** — [Z]が必要なチームに最適
```

---

### 比較インデックス

**URL**：`/vs` または `/compare`

**目的**：すべての「自社 vs [競合]」および「[A] vs [B]」ページを一覧表示する

**ページ構成**：
1. 見出し：「[自社製品]を比較する」
2. セクション：「[自社製品] vs 競合」—直接比較の一覧
3. セクション：「一対一比較」—[A] vs [B] ページの一覧
4. 評価基準の簡単な説明
5. CTA

---

### インデックスページのベストプラクティス

**常に更新する**：新しい比較ページを追加したら、該当インデックスにも追加する。

**内部リンク**：
- インデックス → 個別ページへのリンク
- 個別ページ → インデックスへの逆リンク
- 関連比較ページ同士のクロスリンク

**SEO価値**：
- インデックスページは「プロジェクト管理ツール比較」などの広範なキーワードで上位表示できる
- 個別比較ページへのリンクエクイティを渡せる
- 検索エンジンが全比較コンテンツを効率よく発見できる

**並び順の選択肢**：
- 人気順（検索ボリューム）
- アルファベット順
- カテゴリ/ユースケース別
- 追加日順（新鮮さをアピール）

**インデックスページに含めるもの**：
- 信頼性のための最終更新日
- 掲載ページ数/比較数
- 比較数が多い場合のクイックフィルター

## フッターナビゲーション

サイトフッターは全マーケティングページに表示されるため、競合ページへの内部リンクとして強力な効果がある。

### オプション1：インデックスページへのリンク（最低限）

最低限、フッターに比較インデックスページへのリンクを追加する：

```
フッター
├── 比較
│   ├── 代替ツール →  /alternatives
│   └── 比較一覧 →  /vs
```

これにより、全マーケティングページから比較コンテンツハブへのリンクエクイティが確保される。

### オプション2：形式別フッターカラム（SEO強化に推奨）

より強力な内部リンクを構築するために、形式ごとの専用フッターカラムを作成し、主要競合ページに直接リンクする：

```
フッター
├── [製品] vs               ├── [製品]への代替             ├── 比較
│   ├── vs Notion          │   ├── Notion の代替         │   ├── Notion vs Airtable
│   ├── vs Airtable        │   ├── Airtable の代替       │   ├── Monday vs Asana
│   ├── vs Monday          │   ├── Monday の代替         │   ├── Notion vs Monday
│   ├── vs Asana           │   ├── Asana の代替          │   ├── ...
│   ├── vs Clickup         │   ├── Clickup の代替        │   └── すべて見る →
│   ├── ...                │   ├── ...                  │
│   └── すべて見る →        │   └── すべて見る →          │
```

**ガイドライン**：
- 1カラムにつき最大8リンク（検索ボリュームの高い競合を優先）
- 全インデックスページへの「すべて見る」リンクを追加する
- 実際にページを作成した形式のカラムのみ作成する
- 検索ボリューム最大の競合を優先する

### フッターリンクが重要な理由

1. **サイト全体への配布**：フッターリンクは全マーケティングページに表示され、サイト全体から比較コンテンツへのリンクエクイティが渡される
2. **クロール効率**：検索エンジンが全比較ページを素早く発見できる
3. **ユーザー導線**：製品を評価中の訪問者が簡単に比較ページを見つけられる
4. **競合ポジショニング**：その分野の主要プレイヤーであることを検索エンジンに示せる

### 実装メモ

- 高優先度の新しい比較ページを追加したらフッターを更新する
- フッターはシンプルに保つ—全比較を列挙せず、上位のみ掲載する
- カラムヘッダーをURL構造に合わせる（例：「vs」カラム → `/vs/` URL）
- モバイル対応を考慮：カラムはスタック表示になるため、優先度順に並べる
