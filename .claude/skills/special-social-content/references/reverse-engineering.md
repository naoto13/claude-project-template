# バイラルコンテンツのリバースエンジニアリング

何が機能するかを推測するのではなく、自分のニッチのトップパフォーマンスコンテンツを体系的に分析し、実証済みのパターンを抽出します。

## 目次
- 6 ステップフレームワーク（ニッチ特定、スクレイプ、分析、プレイブック、声の重ね合わせ、コンバート）
- 公式
- リバースエンジニアリングチェックリスト

## 6 ステップフレームワーク

### 1. NICHE ID — トップクリエイターを見つける

自分の分野で一貫して高いエンゲージメントを得ている 10〜20 のクリエイターを特定します：

**選定基準:**
- 一貫して投稿している（週 3 回以上）
- フォロワー数に対して高いエンゲージメント率
- ターゲット市場とのオーディエンスの重複
- 確立されたクリエイターと新興クリエイターのミックス

**見つける場所:**
- LinkedIn: 業界キーワードで検索し、「他にも見られています」を確認
- Twitter/X: ターゲットオーディエンスがフォローしてエンゲージしている人を確認
- SparkToro、Followerwonk などのツールや手動リサーチを使用
- 業界ニュースレターで取り上げられている人を確認

### 2. SCRAPE — 大規模に投稿を収集する

特定したクリエイターから分析用に 500〜1000 件以上の投稿を収集します：

**ツール:**
- **Apify** — LinkedIn スクレイパー、Twitter スクレイパーアクター
- **Phantom Buster** — マルチプラットフォーム自動化
- **エクスポートツール** — プラットフォーム固有のエクスポート機能
- **手動収集** — 小さいデータセットの場合、スプレッドシートにコピー＆ペースト

**収集するデータ:**
- 投稿テキスト/コンテンツ
- エンゲージメントメトリクス（いいね、コメント、シェア、保存）
- 投稿フォーマット（テキストのみ、カルーセル、動画、画像）
- 投稿時間/曜日
- フック/最初の一行
- 使用した CTA
- トピック/テーマ

### 3. ANALYZE — 実際に機能しているものを抽出する

データを並べ替えて分析し、パターンを見つけます：

**定量分析:**
- エンゲージメント率で投稿をランク付けする
- 上位 10% のパフォーマーを特定する
- フォーマットパターンを探す（カルーセルがアウトパフォームするか？）
- タイミングパターンを確認する（最適な曜日/時間帯）
- トピックのパフォーマンスを比較する

**定性分析:**
- 上位の投稿はどんなフックを使っているか？
- パフォーマンスが高い投稿はどのくらいの長さか？
- どんな感情的なトリガーが現れるか？
- どんなフォーマットが繰り返されるか？
- どんなトピックが一貫してパフォーマンスが良いか？

**答えるべき質問:**
- 上位の投稿の平均的な長さは？
- 上位 10% に最もよく現れるフックタイプは？
- どの CTA が最もコメントを促進するか？
- どのトピックが最も保存/シェアされるか？

### 4. PLAYBOOK — パターンを体系化する

使用できる再現可能なパターンを文書化します：

**体系化するフックパターン:**
```
パターン: "I [unexpected action] and [surprising result]"
例: "I stopped posting daily and my engagement doubled"
機能する理由: 好奇心ギャップ + 逆張り

パターン: "[Specific number] [things] that [outcome]:"
例: "7 pricing mistakes that cost me $50K:"
機能する理由: 具体性 + 損失回避

パターン: "[Controversial take]"
例: "Cold outreach is dead."
機能する理由: パターン割り込み + 議論を招く
```

**フォーマットパターン:**
- カルーセル: フックスライド → 問題 → 解決手順 → CTA
- スレッド: フック → 約束 → 届ける → まとめ → CTA
- ストーリー投稿: フック → セットアップ → 葛藤 → 解決 → 教訓

**CTA パターン:**
- 質問: 「何を追加しますか？」
- 同意: 「賛成か反対か？」
- シェア: 「これを必要としている人をタグ付けして」
- 保存: 「後で見るために保存してください」

### 5. LAYER VOICE — ダイレクトレスポンスの原則を適用する

実証済みのパターンを取り入れ、これらのボイス原則で自分のものにします：

**「何かを解明したスマートな友人」**
- 友人にアドバイスをテキストするように書く
- 講義ではなく、発見を共有する
- 「あなたはすべき...」ではなく「私はわかった...」を使う
- 説教がましくなく、役に立つ

**具体的 > 曖昧**
```
❌ "I made good revenue"
✅ "I made $47,329"

❌ "It took a while"
✅ "It took 47 days"

❌ "A lot of people"
✅ "2,847 people"
```

**短く。呼吸する。着地する。**
- 1 文に 1 つのアイデア
- 改行を積極的に使用する
- 重要なポイントを単独で際立たせる
- リズムを作る：短い、短い、長い説明

```
❌ "I spent three years building my business the wrong way before I finally realized that the key to success was focusing on fewer things and doing them exceptionally well."

✅ "I built wrong for 3 years.

Then I figured it out.

Focus on less.
Do it exceptionally well.

Everything changed."
```

**感情から書く**
- 何をしたかではなく、どう感じたかから始める
- 感情的な言葉を使う：frustrated（フラストレーション）、excited（興奮）、terrified（恐怖）、obsessed（執着）
- 本物の場合に脆弱性を見せる
- 感情を教訓と結びつける

```
❌ "Here's what I learned about pricing"

✅ "I was terrified to raise my prices.

My hands were shaking when I sent the email.

Here's what happened..."
```

### 6. CONVERT — 注目をアクションに変える

エンゲージメントをビジネス結果につなげます：

**ソフトコンバージョン:**
- プロフィール/コメントのニュースレター登録
- フォローアップコメントの無料リソース提供
- DM トリガー（「X とコメントすれば送ります...」）
- プロフィール訪問 → 明確な CTA を持つ最適化されたプロフィール

**直接コンバージョン:**
- コメントのリンク（LinkedIn では投稿本文ではなく）
- 価値あるコンテンツ内の文脈的な製品言及
- 自然に自分の仕事を紹介するケーススタディ投稿
- 「これに関して助けが必要なら DM してください」（控えめに）

---

## 公式

```
1. すでに機能しているものを見つける（推測しない）
2. パターンを抽出する（フック、フォーマット、CTA）
3. 上に自分の本物の声を重ねる
4. 自分のデータに基づいてテストして反復する
```
