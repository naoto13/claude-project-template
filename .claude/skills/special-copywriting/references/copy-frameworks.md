## ランディングページのセクションタイプ

### コアセクション

**ヒーロー（ファーストビュー）**
- 見出し + サブヘッドライン
- プライマリCTA
- サポートするビジュアル（製品スクリーンショット、ヒーロー画像）
- オプション：ソーシャルプルーフバー

**ソーシャルプルーフバー**
- 顧客ロゴ（数より認知度）
- 主要指標（「10,000以上のチームが利用」）
- レビュー数付きの星評価
- 短いtestimonialの抜粋

**問題・痛みセクション**
- 彼らの問題を彼ら以上にうまく表現する
- 共感を生む（「まさに自分の状況だ」）
- 解決しないことのコストを示唆する

**解決策・ベネフィットセクション**
- 問題から解決策へのブリッジ
- 主要ベネフィット3〜5個（10個ではなく）
- 各項目：見出し + 説明 + 可能な限り証拠

**仕組み**
- 番号付きの3〜4ステップ
- 複雑さの認知を下げる
- 各ステップ：アクション + 成果

**最終CTAセクション**
- バリュープロポジションの要約
- プライマリCTAの繰り返し
- リスク軽減（保証、無料トライアル）

---

### サポートセクション

**Testimonial**
- 名前・役職・会社付きの完全な引用
- 可能な場合は写真
- 漠然とした賞賛より具体的な結果
- フォーマット：引用カード、動画、Tweetの埋め込み

**ケーススタディ**
- 問題 → 解決策 → 結果
- 具体的な指標と成果
- 顧客名とコンテキスト
- 「詳しく読む」リンク付きのスニペットも可

**ユースケース**
- 製品の様々な使い方
- 訪問者が自分に当てはめるのを助ける
- 「Xを必要とするマーケター向け」の形式

**ペルソナ / 「〜向け」セクション**
- ターゲットオーディエンスを明示する
- 「[役職]に最適」のブロック
- 「これは自分向けか？」という疑問に答える

**FAQセクション**
- よくある反論に対処する
- SEOに効果的
- サポート負担を軽減する
- よくある質問5〜10個

**比較セクション**
- 競合他社と比較（名指しするかしないか）
- 現状（スプレッドシート、手作業）と比較
- テーブルや並列形式

**インテグレーション・パートナー**
- 連携するツールのロゴ
- 「あなたのスタックで動作する」メッセージ
- 信頼性を構築する

**創業者ストーリー・マニフェスト**
- なぜ作ったのか
- 何を信じているか
- 感情的なつながり
- 顔のない競合との差別化

**デモ・製品ツアー**
- インタラクティブデモ
- 動画のウォークスルー
- GIFプレビュー
- 製品の動作を見せる

**料金プレビュー**
- 料金ページ以外でもティーザーとして
- 開始価格または「〇〇円/月から」
- 意思決定者を前進させる

**保証・リスク軽減**
- 返金保証
- 無料トライアルの条件
- 「いつでもキャンセル可能」
- 摩擦を減らす

**統計セクション**
- 信頼性を構築する主要指標
- 「10,000以上の顧客」
- 「4.9/5の評価」
- 「顧客に2億円の節約」

## ページ構造テンプレート

### 機能重視ページ（弱い例）

```
1. ヒーロー
2. 機能1
3. 機能2
4. 機能3
5. 機能4
6. CTA
```

これはリストであり、説得力のあるナラティブではない。

---

### 多様で魅力的なページ（強い例）

```
1. 明確なバリュープロポジションを持つヒーロー
2. ソーシャルプルーフバー（ロゴまたは統計）
3. 問題・痛みセクション
4. 仕組み（3ステップ）
5. 主要ベネフィット（10個ではなく2〜3個）
6. Testimonial
7. ユースケースまたはペルソナ
8. 他の選択肢との比較
9. ケーススタディのスニペット
10. FAQ
11. 保証付きの最終CTA
```

これはストーリーを語り、反論に対処する。

---

### コンパクトなランディングページ

```
1. ヒーロー（見出し、サブヘッド、CTA、画像）
2. ソーシャルプルーフバー
3. アイコン付きの主要ベネフィット3つ
4. Testimonial
5. 仕組み（3ステップ）
6. 保証付きの最終CTA
```

簡潔さが重要な広告ランディングページに適している。

---

### エンタープライズ/B2Bランディングページ

```
1. ヒーロー（成果重視の見出し）
2. ロゴバー（有名企業）
3. 問題セクション（ビジネスの痛み）
4. ソリューション概要
5. 役職・部門別のユースケース
6. セキュリティ・コンプライアンスセクション
7. インテグレーションロゴ
8. 指標付きのケーススタディ
9. ROI・価値セクション
10. 問い合わせ・デモCTA
```

エンタープライズバイヤーの懸念に対処する。

---

### 製品ローンチページ

```
1. ローンチアナウンスを含むヒーロー
2. 動画デモまたはウォークスルー
3. 機能のハイライト（3〜5個）
4. ビフォー/アフターの比較
5. 初期のtestimonial
6. ローンチ価格またはアーリーアクセスオファー
7. 緊急性を持つCTA
```

ProductHunt、ローンチ、アナウンスに適している。

## セクション執筆のコツ

### 問題セクション

以下のようなフレーズで始める:
- 「あの感覚、知ってますか...」
- 「多くの[役職]の方と同じように...」
- 「毎日、[オーディエンス]は...に悩んでいます」
- 「誰でも経験がありますよね...」

その後、以下を描写する:
- 具体的な不満
- 無駄にした時間・お金
- 仕事・生活への影響

### ベネフィットセクション

各ベネフィットに含める:
- **見出し**: 彼らが得る成果
- **本文**: どう機能するか（1〜2文）
- **証拠**: 数字、testimonial、または例（オプション）

### 仕組みセクション

各ステップを:
- **番号付き**: 進捗感を生み出す
- **シンプルな動詞**: 「接続する」「設定する」「取得する」
- **成果指向**: このステップから何を得るか

例:
1. ツールを接続する（2分で完了）
2. 設定をする
3. 毎週月曜日に自動レポートを受け取る

### Testimonialの選定

良いtestimonialに含まれるもの:
- 具体的な結果（「コンバージョンが32%増加した」）
- ビフォー/アフターのコンテキスト（「以前は何時間もかかっていました...」）
- 信頼性のための役職 + 会社名
- 引用可能で具体的な内容

以下のようなtestimonialは避ける:
- 「素晴らしい製品！」
- 「大好きです！」
- 「使いやすい！」
