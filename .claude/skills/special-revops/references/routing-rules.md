# リードルーティングルール

決定ツリー、プラットフォーム固有の設定、テリトリールーティング、ABMルーティング、リード対応速度のベンチマーク。

## ルーティング決定ツリー

このテンプレートを使ってルーティングロジックをマッピングする:

```
新規リードが到着
│
├─ 指名・ターゲットアカウントか？
│  ├─ YES → アサインされたアカウントオーナーへルーティング
│  └─ NO ↓
│
├─ ACVが$50K以上の可能性がある？（会社規模 + 業界に基づく）
│  ├─ YES → エンタープライズAEチームへルーティング
│  └─ NO ↓
│
├─ チーム使用のPLGサインアップか？
│  ├─ YES → PLGセールススペシャリストへルーティング
│  └─ NO ↓
│
├─ リードはテリトリーに一致するか？
│  ├─ YES → テリトリーオーナーへルーティング
│  └─ NO ↓
│
└─ デフォルト: 利用可能な担当者全体でラウンドロビン
   └─ 担当者が利用できない場合: 1時間SLA付きのチームキューにアサイン
```

このツリーをビジネスに合わせてカスタマイズする。重要な原則: **最も具体的なマッチに最初にルーティングし、一般的なものにフォールバックする。**

## ラウンドロビン設定

### 基本的なラウンドロビンルール

1. 対象の担当者全員に均等にリードを配分する
2. 休暇中、キャパシティ超過、または満杯のパイプラインを持つ担当者はスキップする
3. クォータ達成率で重み付けする（クォータ未達の担当者に若干優先）
4. 配分カウントを週次または月次でリセットする
5. 監査のためにすべてのアサインをログに記録する

### HubSpotのラウンドロビン設定

**HubSpotのローテーションツールを使用:**
- オートメーション → ワークフローに移動
- トリガー: コンタクトプロパティ「ライフサイクルステージ」が「MQL」
- アクション: 選択したユーザー間でコンタクトオーナーをローテーション
- オプション: 均等配分、利用できないオーナーをスキップ
- アサイン後に遅延 + タスク作成を追加

**ワークフローによるカスタムローテーション:**
1. カスタムプロパティ「ローテーションカウンター」（数値）を作成する
2. ワークフロートリガー: 新しいMQLが作成された時
3. ローテーションカウンター値（各担当者に対して0、1、2...）で分岐
4. 対応する担当者にコンタクトオーナーを設定する
5. カウンターをインクリメント（最大値でリセット）
6. SLA期限付きのフォローアップタスクを作成する

### Salesforceのラウンドロビン設定

**リードアサインメントルールを使用:**
1. 設定 → 機能設定 → マーケティング → リードアサインメントルール
2. 優先度順にルールエントリを作成する（最も具体的なものを最初に）
3. ラウンドロビンには: アサインメントルール + カスタムロジックを使用

**高度なルーティングにFlowを使用:**
1. Lead作成時にレコードトリガーFlowを作成する
2. レコード取得: 次の利用可能な担当者を「Rep Queue」カスタムオブジェクトからクエリ
3. 判断要素: 担当者の可用性、キャパシティ、テリトリーを確認
4. レコード更新: リードオーナーをアサイン
5. タスクを作成: SLA付きのフォローアップタスク
6. 「Rep Queue」を更新して最終アサインをトラック

## テリトリールーティング

### 地域別

| テリトリー | 地域 | アサインされたチーム |
|-----------|---------|---------------|
| 西部 | CA、WA、OR、NV、AZ、UT、CO、HI | チーム西部 |
| 中部 | TX、IL、MN、MO、OH、MI、WI、IN | チーム中部 |
| 東部 | NY、MA、PA、NJ、CT、VA、FL、GA | チーム東部 |
| インターナショナル | 米国以外すべて | インターナショナルチーム |

### 会社規模別

| セグメント | 会社規模 | チーム |
|---------|-------------|------|
| SMB | 従業員1〜50 | インサイドセールス |
| ミッドマーケット | 従業員51〜500 | ミッドマーケットAE |
| エンタープライズ | 従業員501〜5000 | エンタープライズAE |
| ストラテジック | 従業員5000以上 | 戦略的アカウントチーム |

### 業界別

| バーティカル | 業界 | スペシャリスト |
|----------|-----------|------------|
| テック | SaaS、ITサービス、ハードウェア | テックバーティカル担当者 |
| 金融 | 銀行、保険、フィンテック | 金融バーティカル担当者 |
| ヘルスケア | 病院、製薬、ヘルステック | ヘルスケアバーティカル担当者 |
| 一般 | その他すべて | 一般プール（ラウンドロビン） |

### ハイブリッドテリトリーモデル

複数の次元を組み合わせて精度を高める:

```
リードが到着
├─ 会社規模 > 1000か？
│  ├─ YES → エンタープライズチーム
│  │  └─ 地域でサブルーティング
│  └─ NO ↓
├─ 業界 = ヘルスケアまたは金融？
│  ├─ YES → バーティカルスペシャリスト
│  └─ NO ↓
└─ 一般プールでラウンドロビン
   └─ 地域の優先度で重み付け
```

## 指名アカウント / ABMルーティング

### セットアップ

1. **ターゲットアカウントリストを定義する**（通常50〜500アカウント）
2. **CRMでアカウントオーナーをアサインする**（アカウントごとに1担当者）
3. **マッチングロジック:** ターゲットアカウントドメインからのリードはアカウントオーナーへルーティング
4. **マッチングルール:**
   - メールドメインマッチ（主要）
   - 会社名のファジーマッチ（二次的、手動レビューが必要）
   - IPから企業への解決（三次的、匿名訪問者向け）

### ABMルーティングルール

| 階層 | アカウントタイプ | ルーティング | 応答SLA |
|------|-------------|---------|--------------|
| Tier 1 | 戦略的上位20アカウント | 指名オーナー、即時アラート | 1時間 |
| Tier 2 | ターゲット上位100アカウント | 指名オーナー、標準アラート | 4時間 |
| Tier 3 | ターゲット業界・規模マッチ | テリトリーまたはラウンドロビン | 当営業日内 |

### マルチコンタクトの処理

同じアカウントから複数のコンタクトがエンゲージした場合:
- すべてのコンタクトを**同じアカウントオーナー**にルーティングする
- 新しいコンタクトが参入した際にオーナーに通知する
- アカウントレベルのエンゲージメントスコアを追跡する（全コンタクトの合算）
- 1つのアカウントから3つ以上のコンタクトがエンゲージした場合に「バイイングコミッティ」アラートをトリガー

## リード対応速度のデータ

### 対応時間がコンバージョンに与える影響

| 対応時間 | 相対的な資格確認率 | メモ |
|---------------|---------------------------|-------|
| 5分以内 | **21倍**多く資格確認される | ゴールドスタンダード |
| 5〜10分 | 10倍多い | まだ強力 |
| 10〜30分 | 4倍多い | ほとんどの場合で許容範囲 |
| 30分〜1時間 | 2倍多い | ベストプラクティス以下 |
| 1〜24時間 | ベースライン | 業界平均 |
| 24時間以上 | ベースラインより60%低い | リードは事実上冷却 |

出典: Lead Connect、InsideSales.com

### リード対応速度の実装

1. **即時通知** — MQL作成時にプッシュ通知 + 担当者へメール
2. **タイマー付き自動タスク** — 5分SLAカウントダウン付きのタスクを作成
3. **エスカレーションチェーン:**
   - 5分: 元の担当者にアラート
   - 15分: バックアップ担当者にアラート
   - 30分: マネージャーにアラート
   - 1時間: 次の利用可能な担当者に再アサイン
4. **測定とレポート** — 実際の対応時間を週次で追跡。迅速な対応者を表彰

### リード対応速度のオートメーション

**トリガー:** 新しいMQLが作成された時
**アクション:**
1. ルーティングルールで担当者にアサイン（即時）
2. 担当者にプッシュ通知 + メールを送る
3. タスクを作成: 「[リード名]に連絡 — 5分SLA」
4. SLAタイマーを開始
5. 15分以内に活動がログされない場合 → バックアップ担当者にアラート
6. 30分以内に活動がない場合 → マネージャーにアラート
7. 60分以内に活動がない場合 → ラウンドロビンで再アサイン

### リード対応速度の測定

以下の指標を週次で追跡する:
- **初回連絡までの平均時間**（MQL作成から初回通話・メールまで）
- **初回連絡までの中央値時間**（外れ値の影響が少ない）
- **SLA内に連絡したリードの割合**（目標: 90%以上）
- **時間帯別の連絡率**（カバレッジのギャップを特定）
- **対応時間別コンバージョン率**（対応速度のROIを証明）
