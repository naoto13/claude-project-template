# オートメーションプレイブック

HubSpot、Salesforce、スケジューリングツール、クロスツールオートメーションのプラットフォーム固有ワークフローレシピ。

## HubSpotワークフローレシピ

### 1. MQLアラートとアサイン

**名前:** MQL通知とタスク作成
**トリガー:** コンタクトプロパティ「ライフサイクルステージ」が「Marketing Qualified Lead」に変更された時
**アクション:**
1. セールスチーム間でコンタクトオーナーをラウンドロビンで回す
2. コンタクトオーナーにリードコンテキスト付きの内部メール通知を送る
3. タスクを作成: 「[コンタクト名] にフォローアップ」— 4時間以内期限
4. #sales-alertsチャンネルにSlack通知を送る
5. 「MQLフォローアップ」シーケンスに登録（HubSpot Sequencesを使用している場合）
**成果:** すべてのMQLが即座にアサインされ、明確なSLAが設定される
**注記:** 既にrepに担当者がいるリードは除外するよう登録基準を設定する

---

### 2. MQL SLAエスカレーション

**名前:** MQL SLA違反アラート
**トリガー:** コンタクトプロパティ「ライフサイクルステージ」が「MQL」で、「最終連絡からの日数」が0.5（12時間）を超えた時
**アクション:**
1. コンタクトオーナーに内部メールを送る: 「SLA警告: [コンタクト名]にまだ連絡していません」
2. 24時間経っても活動がない場合 → セールスマネージャーにアラートを送る
3. 48時間経っても活動がない場合 → ローテーションでコンタクトオーナーを再アサイン
4. 新しいオーナーにタスクを作成: 「緊急: [コンタクト名]に連絡 — SLA違反により再アサイン」
**成果:** MQLが48時間以上未対応になることがなくなる
**注記:** 最終活動タイプが「通話」または「ミーティング」のコンタクトは除外する（既に対応中）

---

### 3. リードスコア更新とMQLへの昇格

**名前:** スコア閾値で自動MQL化
**トリガー:** コンタクトプロパティ「HubSpotスコア」が65以上になった時
**アクション:**
1. ライフサイクルステージを「Marketing Qualified Lead」に設定
2. 「MQL日付」を現在の日付に設定
3. マーケティングナーチャリングワークフローから除外
4. MQLアラートワークフロー（レシピ#1）をトリガー
**成果:** リードがスコアリング閾値に達すると自動的にMQLに昇格
**注記:** 既存顧客と競合他社の除外リストを追加する

---

### 4. ミーティング予約通知

**名前:** ミーティング予約のAEへのアラート
**トリガー:** コンタクトにミーティングアクティビティが記録された時（Calendly/HubSpotミーティング経由）
**アクション:**
1. ミーティング詳細付きでコンタクトオーナーに内部メールを送る
2. コンタクトプロパティ「最終ミーティング予約日」を現在の日付に更新
3. ライフサイクルステージが「Lead」の場合 → 「MQL」に更新
4. タスクを作成: 「[コンタクト名]とのミーティング準備」— ミーティング1時間前期限
5. #meetingsチャンネルにSlack通知を送る
**成果:** AEがすべてのミーティングに完全なコンテキストで臨める
**注記:** 通知メールに最近のページビューとコンテンツダウンロードを含める

---

### 5. クローズドウォン時のCSへの引き継ぎ

**名前:** カスタマーオンボーディングトリガー
**トリガー:** 案件ステージが「Closed Won」に変更された時
**アクション:**
1. 関連コンタクトのライフサイクルステージを「Customer」に更新
2. 「顧客獲得日」を現在の日付に設定
3. セグメント・テリトリーに基づいてCSチームメンバーにコンタクトオーナーをアサイン
4. CSにタスクを作成: 「[会社名]とのキックオフコールをスケジュール」— 2営業日以内期限
5. コンタクトを「カスタマーオンボーディング」メールシーケンスに登録
6. CSマネージャーに内部通知を送る
7. すべてのセールスシーケンスから削除
**成果:** セールスからカスタマーサクセスへのシームレスな引き継ぎ
**注記:** CS通知に案件メモ、契約額、主要ステークホルダーを含める

---

### 6. 滞留案件アラート

**名前:** パイプライン衛生 — 滞留案件の検出
**トリガー:** 案件プロパティ「現在ステージの日数」が[そのステージの平均の2倍]を超えた時
**アクション:**
1. 案件オーナーに内部メールを送る: 「案件滞留アラート: [案件名]が[ステージ]に[X]日間留まっています」
2. タスクを作成: 「[案件名]を更新またはクローズ」— 3営業日以内期限
3. 7日後も更新がない場合 → セールスマネージャーにアラート
4. 「滞留案件」ダッシュボードリストに追加
**成果:** パイプラインが整理され、予測の精度が維持される
**注記:** ステージごとに閾値をカスタマイズする（Discovery: 14日、Proposal: 10日、Negotiation: 21日）

---

### 7. リサイクルリードのナーチャリング再登録

**名前:** MQLリサイクルによるナーチャリング
**トリガー:** コンタクトプロパティ「セールス却下理由」が入力された時（任意の値）
**アクション:**
1. ライフサイクルステージを「Recycled」に更新
2. エンゲージメントスコアをベースラインにリセット（フィットスコアは保持）
3. 「リサイクルリードナーチャリング」シーケンスに登録（頻度を下げる）
4. 「リサイクル日付」を現在の日付に設定
5. 再登録トリガーを設定: HubSpotスコアが再度閾値を超えた場合、MQLワークフローを再トリガー
**成果:** 却下されたリードがパイプラインを詰まらせることなくセカンドチャンスを得る
**注記:** リサイクル→MQLコンバージョン率を別途追跡する

---

### 8. リードアクティビティダイジェスト

**名前:** 日次リードアクティビティサマリー
**トリガー:** スケジュール — 毎日現地時間の午前8時
**アクション:**
1. コンタクトをフィルタ: ライフサイクルステージが「SQL」または「Opportunity」で、過去24時間にウェブサイト活動がある
2. 各コンタクトオーナーに担当リードのアクティビティダイジェストメールを送る
3. 含める内容: 訪問ページ、ダウンロードしたコンテンツ、開封・クリックされたメール
**成果:** セールス担当者が毎朝どのリードがアクティブかを把握できる
**注記:** 意味のある活動のあるリードのみ含める（ホームページへの単一訪問は除外）

## Salesforce Flow の対応レシピ

### 1. MQLアラートとアサイン（Salesforce Flow）

**タイプ:** レコードトリガーFlow
**オブジェクト:** Lead
**トリガー:** リードフィールド「Status」が「MQL」に変更された時
**フローステップ:**
1. レコード取得: 「Rep Assignment」カスタムオブジェクトから次の利用可能な担当者をクエリ
2. レコード更新: Lead所有者をアサインされた担当者に設定
3. レコード作成: タスクを作成 — 「Contact MQL: {Lead.Name}」期限 = NOW + 4時間
4. アクション: 新しいリードオーナーにメールアラートを送る
5. レコード更新: 「Rep Assignment」の最終アサインタイムスタンプを更新
**注記:** ラウンドロビン状態を管理するためにカスタム「Rep Assignment」オブジェクトを使用する

### 2. SLAエスカレーション（Salesforce Flow）

**タイプ:** スケジュールトリガーFlow
**スケジュール:** 営業時間中は4時間ごと
**フローステップ:**
1. レコード取得: Status = "MQL" で LastActivityDate < TODAY - 1 のリード
2. 判断: 活動なしで48時間を超えているか？
   - YES → 次の担当者に再アサイン、緊急タスクを作成、マネージャーにアラート
   - NO → 現在のオーナーにリマインダーメールを送る
**注記:** 初期アサイン時のリアルタイムアラートにはProcess Builderと組み合わせる

### 3. パイプラインステージオートメーション（Salesforce Flow）

**タイプ:** レコードトリガーFlow
**オブジェクト:** Opportunity
**トリガー:** Stageフィールドが更新された時
**フローステップ:**
1. 判断: どのステージに変更されたか？
2. 各ステージについて:
   - **Discovery:** タスクを作成「ディスカバリー質問票を完了」
   - **Demo:** タスクを作成「デモ環境を準備」
   - **Proposal:** タスクを作成「提案書を送付」+ ACVが25,000ドル超の場合はディールデスクにアラート
   - **Closed Won:** CSへの引き継ぎをトリガー（ケースを作成、CSオーナーをアサイン、ウェルカムメールを送信）
   - **Closed Lost:** タスクを作成「失注理由を記録」+ 勝敗分析レポートに追加

### 4. 滞留案件の検出（Salesforce Flow）

**タイプ:** スケジュールトリガーFlow
**スケジュール:** 毎日午前7時
**フローステップ:**
1. レコード取得: Days_In_Stage > Stage_SLA_Thresholdのオープン商談
2. 結果をループ:
   - タスクを作成: 「滞留案件を更新: {Opportunity.Name}」
   - Opportunity所有者にメールを送る
   - Days_In_Stageが閾値の2倍を超える場合 → 所有者のマネージャーにメールを送る
3. カスタムフィールド「Stale Flag」= trueに更新（ダッシュボードの可視性のため）

## Calendly / SavvyCal インテグレーションパターン

### ラウンドロビンミーティングスケジューリング

**Calendlyのセットアップ:**
1. すべての対象担当者を含むチームイベントタイプを作成する
2. 配分: 「均等配分を最適化」
3. 可用性: 各担当者が自分のカレンダーを管理する
4. バッファ: ミーティングの前後15分
5. 最小通知: 4時間（直前の予約を避ける）

**CRMインテグレーション:**
1. CalendlyのWebhookが予約時に起動する
2. 招待者のメールをCRMコンタクトに一致させる
3. コンタクトが存在する場合 → コンタクトオーナーにミーティングをアサイン（ラウンドロビンをオーバーライド）
4. 新しいコンタクトの場合 → リードを作成し、ルーティングルールでアサイン、ミーティングをログ
5. ライフサイクルステージをMQLに設定（ミーティング = 高い意図）

### SavvyCal セットアップ

**Calendlyに対する優位点:**
- 優先度ベースのスケジューリング（特定の時間枠を優先）
- カレンダーのオーバーレイ（1つのビューでチームの可用性を表示）
- 担当者ごとのパーソナライズされた予約リンク

**インテグレーションパターン:**
1. 優先度ルール付きのチームスケジューリングリンクを作成する
2. 予約時のWebhook → Zapier/Make → CRM
3. コンタクトを一致または作成し、オーナーをアサイン、タスクを作成
4. ミーティング準備資料付きの確認を送る

### 基準によるミーティングルーティング

```
予約フォームが送信された
├─ 従業員数 > 500？（フォームフィールド）
│  ├─ YES → エンタープライズAEカレンダーへルーティング
│  └─ NO ↓
├─ 既存顧客？（CRMルックアップ）
│  ├─ YES → アカウントオーナーのカレンダーへルーティング
│  └─ NO ↓
└─ SDRチーム全体でラウンドロビン
```

### ノーショーワークフロー

**トリガー:** ミーティング時間経過後、30分以内にミーティングノートが記録されなかった時
**アクション:**
1. 予定ミーティング時間から30分待つ
2. 確認: 通話またはミーティングがログに記録されたか？
   - YES → アクションなし
   - NO → 見込み客に「お会いできなかったようです」メールを送る
3. タスクを作成: 「[コンタクト名]と再スケジュール」— 次の営業日期限
4. 2度目のノーショーの場合 → コンタクントをフラグし、マネージャーにアラート

## Zapier クロスツールパターン

### 1. 新規リード → CRM + Slack + タスク

**トリガー:** 新しいフォーム送信（Typeform、HubSpot、Webflow）
**アクション:**
1. CRMでコンタクトを作成または更新
2. Clearbitでエンリッチ（利用可能な場合）
3. エンリッチされたデータをSlack #new-leadsにポスト
4. プロジェクト管理ツールでタスクを作成（Asana、Linear）

### 2. ミーティング予約 → CRM + 準備メール

**トリガー:** 新しいCalendly/SavvyCal予約
**アクション:**
1. CRMコンタクトを検索または作成
2. ライフサイクルステージをMQLに更新
3. アサインされた担当者に準備メールを送る（CRMリンク、LinkedInプロフィール、最近の活動を含む）
4. ミーティング前タスクを作成

### 3. 案件クローズ → オンボーディングスタック

**トリガー:** CRMの案件ステージが「Closed Won」に変更された時
**アクション:**
1. CSツール（Vitally、Gainsight、ChurnZero）で顧客レコードを作成
2. オンボーディングプロジェクトテンプレートに追加
3. メールツール経由でウェルカムメールを送る
4. Slackチャンネルを作成: #customer-[会社名]
5. SlackでCSチームに通知

### 4. リードスコアリング → クロスツール同期

**トリガー:** CRMリードスコアがMQL閾値を超えた時
**アクション:**
1. マーケティングオートメーションプラットフォームのステータスを更新
2. リターゲティングオーディエンスに追加（Facebook、Google Ads）
3. SDRアウトリーチシーケンスをトリガー
4. アナリティクスにイベントをログ（Mixpanel、Amplitude）

### 5. SLA違反 → マルチチャンネルアラート

**トリガー:** CRMタスクが期限切れ（MQLフォローアップタスク）
**アクション:**
1. 担当者にSlack DMを送る
2. 担当者にメールを送る
3. 2時間以上経過 → マネージャーにSlack DMを送る
4. 4時間以上経過 → CRMで再アサイン（CRMへのWebhook経由）

### 6. 週次パイプラインダイジェスト

**トリガー:** スケジュール — 毎週月曜日午前8時
**アクション:**
1. CRMからパイプラインサマリーをクエリ（総額、新規案件、滞留案件、期待されるクローズ）
2. サマリーとしてフォーマット
3. Slack #sales-teamにポスト
4. セールスリーダーシップにメールダイジェストを送る
