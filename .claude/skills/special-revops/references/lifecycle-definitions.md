# ライフサイクルステージ定義

リードライフサイクルステージの完全なテンプレート、ビジネスタイプ別MQL基準、SLA、および却下・リサイクルワークフロー。

## ステージテンプレート

### サブスクライバー

**参入基準:**
- ブログ、ニュースレター、またはコンテンツ更新にオプトイン
- 会社情報は不要

**脱出基準:**
- フォームまたはエンリッチメントで会社情報を提供する
- 1セッションで3ページ以上を訪問する
- ゲートコンテンツをダウンロードする

**担当者:** マーケティング（自動）

**参入時のアクション:**
- ニュースレターナーチャリングに追加
- エンゲージメントスコアのトラッキングを開始

---

### リード

**参入基準:**
- 名前 + メール + 会社名を持つ特定済みのコンタクト
- フォーム入力、エンリッチメント、またはインポートから来る場合がある

**脱出基準:**
- MQL閾値に達する（フィット + エンゲージメント）
- マーケティング・SDRが手動で資格確認する

**担当者:** マーケティング

**参入時のアクション:**
- コンタクトデータをエンリッチ（会社規模、業界、役割）
- スコアリングを開始
- 関連するナーチャリングシーケンスに追加

---

### MQL（Marketing Qualified Lead）

**参入基準:**
- フィットスコア閾値とエンゲージメントスコア閾値の両方を満たす
- または高意図アクションをトリガーする（デモリクエスト、価格ページ + フォーム入力）

**脱出基準:**
- セールスが承認（SQLになる）
- セールスが却下（理由コード付きでナーチャリングにリサイクル）
- SLA内に応答なし（マネージャーにエスカレーション）

**担当者:** マーケティング → セールス（引き継ぎ）

**参入時のアクション:**
- アサインされたセールス担当者に即座にアラート
- 4時間SLA付きのフォローアップタスクを作成
- マーケティングナーチャリングシーケンスを一時停止
- セールスのコンテキスト用に最近の全アクティビティをログ

---

### SQL（Sales Qualified Lead）

**参入基準:**
- セールス担当者が資格確認の会話をした
- 予算、権限、ニーズ、またはタイムラインのうち少なくとも2つを確認

**脱出基準:**
- 予測額を含む商談が作成される
- 失格（理由コード付きでリサイクル）

**担当者:** セールス（SDRまたはAE）

**参入時のアクション:**
- CRMでライフサイクルステージを更新
- SDR資格確認の場合はAEに通知
- 会話中でない場合はセールスシーケンスを開始

---

### 商談（Opportunity）

**参入基準:**
- CRMで正式な商談が作成される
- 案件額、クローズ日、ステージがアサインされる

**脱出基準:**
- クローズドウォンまたはクローズドロスト

**担当者:** セールス（AE）

**参入時のアクション:**
- パイプラインレポートに追加
- 案件タスクを作成（提案書、デモ等）
- 案件がクローズしそうな場合はCSに通知

---

### 顧客（Customer）

**参入基準:**
- クローズドウォン案件
- 契約書に署名し、支払い条件が設定される

**脱出基準:**
- チャーン、拡大、または更新

**担当者:** カスタマーサクセス / アカウント管理

**参入時のアクション:**
- オンボーディングシーケンスをトリガー
- CSマネージャーをアサイン
- キックオフコールをスケジュール
- すべてのセールスシーケンスから削除

---

### エバンジェリスト（Evangelist）

**参入基準:**
- NPS 9〜10、またはアクティブな紹介行動
- ケーススタディ、推薦文、または紹介プログラムに同意

**脱出基準:**
- 継続的なプログラム参加

**担当者:** カスタマーサクセス + マーケティング

**参入時のアクション:**
- アドボカシープログラムに追加
- ケーススタディまたは推薦文をリクエスト
- 紹介プログラムに招待
- マーケティングキャンペーンでフィーチャー（許可を得て）

---

## ビジネスタイプ別MQL基準テンプレート

### PLG（プロダクトレッドグロース）

**フィットスコア（40%ウェイト）:**

| 属性 | ポイント |
|-----------|--------|
| 会社規模10〜500 | +15 |
| 会社規模500〜5000 | +20 |
| ターゲット業界 | +10 |
| 意思決定者の役職 | +15 |
| 補完ツールを使用 | +10 |

**エンゲージメントスコア（60%ウェイト）— プロダクト使用を重視:**

| シグナル | ポイント |
|--------|--------|
| 無料アカウントを作成 | +15 |
| オンボーディングを完了 | +20 |
| コア機能を3回以上使用 | +25 |
| チームメンバーを招待 | +20 |
| 使用制限に達した | +15 |
| 価格ページを訪問 | +10 |

**MQL閾値:** 65ポイント

## SLAテンプレート

### MQL→SQL SLA

| 指標 | 目標 | エスカレーション |
|--------|--------|------------|
| 初回連絡試行 | 4営業時間以内 | 4時間でセールスマネージャーにアラート |
| 資格確認の判断 | 48時間以内 | 48時間で自動エスカレーション |
| ミーティングのスケジュール（資格確認の場合） | 5営業日以内 | 週次パイプラインレビューフラグ |

### SQL→商談 SLA

| 指標 | 目標 | エスカレーション |
|--------|--------|------------|
| ディスカバリーコールの完了 | SQL後3営業日以内 | AEマネージャーへアラート |
| 商談の作成 | SQL後5営業日以内 | パイプラインレビューフラグ |

### 商談→クローズ SLA

| 指標 | 目標 | エスカレーション |
|--------|--------|------------|
| 提案書の送付 | デモ後5営業日以内 | AEマネージャーアラート |
| ステージでの案件滞留 | そのステージの平均日数の2倍 | パイプラインレビューフラグ |
| クローズ日が2回以上変更 | 即時 | 予測レビューが必要 |

## リード却下とリサイクル

### 却下理由コード

| コード | 理由 | リサイクルアクション |
|------|--------|----------------|
| **FIT-01** | 会社規模が小さすぎる | ナーチャリング。会社が成長した場合に再スコア |
| **FIT-02** | 業界が違う | アーカイブ。リサイクルしない |
| **FIT-03** | 役職が違う・権限なし | ナーチャリング。組織変更をモニタリング |
| **ENG-01** | 3回試みて応答なし | 90日後にナーチャリングにリサイクル |
| **ENG-02** | 興味はあるが時期が悪い | ナーチャリングにリサイクル。60日後に再エンゲージ |
| **QUAL-01** | 予算なし | 90日後にナーチャリングにリサイクル |
| **QUAL-02** | 競合を使用中、ロックイン | リサイクル。契約更新前にトリガー |
| **QUAL-03** | 実際のプロジェクトではない | アーカイブ。リサイクルしない |

### リサイクルワークフロー

1. セールスがMQLを理由コード付きで却下
2. CRMがライフサイクルステージを「Recycled」に更新
3. リードがリサイクルナーチャリングシーケンスに入る（元のナーチャリングとは異なる）
4. エンゲージメントスコアがベースラインにリセット（フィットスコアは保持）
5. リードが再エンゲージしてMQL閾値を超えた場合、「リサイクルMQL」フラグ付きでセールスに再ルーティング
6. リサイクルMQLコンバージョン率を別途追跡

### リサイクルナーチャリングシーケンス

- **頻度:** 隔週または月次（初回ナーチャリングより低頻度）
- **コンテンツ:** 業界インサイト、ケーススタディ、プロダクト更新
- **期間:** 6ヶ月。エンゲージメントがない場合はアーカイブ
- **MQL再トリガー:** 高意図アクション（デモリクエスト、価格ページの再訪問）
