このスキルは、汎用的な「AIっぽい」見た目を避けた、独創的で本番グレードのフロントエンドインターフェース作成をガイドする。美的ディテールとクリエイティブな選択に細心の注意を払った、実際に動作するコードを実装する。

ユーザーはフロントエンドの要件を提示する：コンポーネント、ページ、アプリケーション、またはインターフェース。目的、ターゲットユーザー、技術的制約についてのコンテキストが含まれる場合もある。

## デザイン思考

コーディングの前に、コンテキストを理解し、**大胆な**美的方向性にコミットする：
- **目的**：このインターフェースはどんな問題を解決するか？誰が使うか？
- **トーン**：極端なものを選ぶ：徹底的にミニマル、マキシマリストの混沌、レトロフューチャー、有機的/自然、ラグジュアリー/洗練、遊び心/トイ風、エディトリアル/マガジン、ブルータリスト/ロー、アールデコ/ジオメトリック、ソフト/パステル、インダストリアル/実用的など。多様な選択肢からインスピレーションを得つつ、美的方向性に忠実なデザインを生み出す。
- **制約**：技術要件（フレームワーク、パフォーマンス、アクセシビリティ）。
- **差別化**：何がこれを**忘れられない**ものにするか？誰もが記憶に残る一つのポイントは？

**重要**：明確なコンセプトの方向性を選び、精緻に実行する。大胆なマキシマリズムも洗練されたミニマリズムも有効 — 鍵は強度ではなく意図性。

### デザイントークンを先に定義する

コード生成前に、以下のプリセット構造を決定する：

| カテゴリ | 定義するもの |
|---|---|
| 背景色 | `background` / `surface` トークン |
| テキスト色 | `foreground` / `muted` / `accent` トークン |
| タイポグラフィロール | `display` / `heading` / `body` / `caption` の4段階 |
| アクセントカラー | ブランドカラーの明示的指定 |

CSS カスタムプロパティまたは Tailwind の `theme.extend` でトークンを実装し、コード全体で一貫して参照する。

実装する動作コード（HTML/CSS/JS、React、Vue等）は：
- 本番グレードで機能的
- 視覚的にインパクトがあり記憶に残る
- 明確な美的視点で統一感がある
- あらゆるディテールが入念に仕上げられている

## ナラティブ構造

ページ全体をストーリーとして設計する。単一のビジュアルアンカーを設け、明確な視覚的階層を維持する：

1. **ヒーロー** — 一目で世界観を伝える。画像・タイポグラフィ・モーションの最大投資ポイント
2. **サポートビジュアル** — ヒーローの印象を補強する画像やイラスト
3. **プロダクト詳細** — 機能・特徴を具体的に伝える
4. **社会的証明** — テスティモニアル、ロゴ、数値実績
5. **最終CTA** — 明確なアクションで締める

すべてのセクションが同じ重みでは退屈になる。**コントラストとリズム**で緩急をつける。

## フロントエンド美学ガイドライン

以下に注力する：
- **タイポグラフィ**：美しく、ユニークで、興味深いフォントを選ぶ。ArialやInterのような汎用フォントを避け、フロントエンドの美学を高める独創的な選択をする。意外性のある個性的なフォントを。特徴的なディスプレイフォントと洗練されたボディフォントをペアリングする。タイポグラフィロールを `display` / `heading` / `body` / `caption` の4段階で統一する。
- **カラー＆テーマ**：統一感のある美学にコミットする。一貫性のためCSSカスタムプロパティを使用。控えめで均等に配分されたパレットよりも、ドミナントカラーに鋭いアクセントを効かせる方が効果的。
- **モーション**：最低**3つの意図的なアニメーション**を実装する — (1) ヒーロー登場アニメーション (2) スクロール連動エフェクト (3) ホバー遷移。ノイズではなく「存在感」のためにモーションを使う。HTMLではCSSのみのソリューションを優先。ReactではMotionライブラリを利用可能なら使用。散在するマイクロインタラクションより、スタガード表示（animation-delay）を用いた統一感のあるページロードの方が喜びを生む。
- **空間構成**：意外性のあるレイアウト。非対称。オーバーラップ。対角線の流れ。グリッドを破る要素。余白を贅沢に、または密度をコントロール。
- **背景＆ビジュアルディテール**：ソリッドカラーにデフォルトせず、雰囲気と奥行きを作る。全体の美学に合ったコンテキストに即したエフェクトやテクスチャを追加。グラデーションメッシュ、ノイズテクスチャ、ジオメトリックパターン、レイヤード透過、ドラマチックなシャドウ、装飾ボーダー、カスタムカーソル、グレインオーバーレイなどのクリエイティブな形を適用。

汎用的なAI生成美学を**絶対に使わない**：使い古されたフォントファミリー（Inter、Roboto、Arial、システムフォント）、陳腐なカラースキーム（特に白背景に紫グラデーション）、予測可能なレイアウトやコンポーネントパターン、コンテキスト固有の個性に欠けるテンプレ的デザイン。

### ハードルール

- **カードをデフォルトにしない** — ジェネリックな SaaS カードグリッドは没個性の象徴。カードが本当に最適な場合のみ使用し、代わりにフルブリード画像、テキストコラム、オーバーラップレイアウト等を検討する
- **ヒーローカードをデフォルトにしない** — ヒーローは画面全体を使って世界観を伝える。カード内にヒーローを閉じ込めない
- **プレースホルダーテキストを使わない** — Lorem ipsum を避け、実際のコンテンツまたはリアリスティックなダミーコンテンツを使用する
- **オーバーエンジニアリングしない** — シンプルな構成にはシンプルなコードを。React Compiler 環境では `useMemo`/`useCallback` をデフォルトで追加しない（Compiler が自動最適化する）

クリエイティブに解釈し、コンテキストに合った本物のデザインに感じられる意外な選択をする。同じデザインは二度とない。ライト/ダークテーマ、異なるフォント、異なる美学を使い分ける。世代を超えて共通の選択（例：Space Grotesk）に**絶対に収束しない**。

**重要**：実装の複雑さを美的ビジョンに合わせる。マキシマリストデザインには豊富なアニメーションとエフェクトを備えた精巧なコードが必要。ミニマリストや洗練されたデザインには抑制、精密さ、スペーシング・タイポグラフィ・繊細なディテールへの注意が必要。エレガンスはビジョンを的確に実行することから生まれる。

Claude は並外れたクリエイティブワークが可能であることを忘れずに。枠にとらわれず考え、独創的なビジョンに全力でコミットすることで何が生み出せるかを示す。
