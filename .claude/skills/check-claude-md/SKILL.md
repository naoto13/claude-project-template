# CLAUDE.md 品質チェッカー

記事「AI-Ready リポジトリ設計ガイド」に基づく15項目チェックリストで CLAUDE.md の品質を診断する。

## ワークフロー

### Phase 0: 対象ファイル特定

1. `$ARGUMENTS` にパスがあればそのファイルを読む
2. なければプロジェクトルートの `CLAUDE.md` → `AGENTS.md` → `agent.md` の順で検索
3. 見つからなければ「ファイルが存在しません。`/claude-md-gen` で生成してください」と報告して終了

### Phase 1: ドキュメント品質チェック（5項目）

対象ファイルを Read で読み込み、以下を判定する:

| # | チェック項目 | 合格基準 | 重み |
|---|---|---|---|
| D1 | CLAUDE.md / AGENTS.md が存在する | ファイルが存在し、空でない | 必須 |
| D2 | 技術スタックが記載されている | 言語・フレームワーク・主要ライブラリが明記 | 必須 |
| D3 | NEVER セクションがある | 「NEVER」「禁止」「絶対やらない」等のセクションが存在し、3項目以上 | 必須 |
| D4 | ワンコマンドセットアップ | セットアップ手順が記載され、2ステップ以内で開発開始可能 | 推奨 |
| D5 | コード実例が2個以上 | コードブロック（```）が2つ以上あり、プロジェクト固有のパターンを示している | 推奨 |

### Phase 2: リポジトリ構造チェック（5項目）

Glob / Grep / Read でリポジトリを調査し、以下を判定する:

| # | チェック項目 | 合格基準 | 重み |
|---|---|---|---|
| S1 | 機能単位のディレクトリ分割 | `features/`, `modules/`, または Vertical Slice 構造がある | 推奨 |
| S2 | 型定義にコメントがある | 主要な型定義ファイルに JSDoc / コメントが付いている（3ファイル以上サンプリング） | 推奨 |
| S3 | 構造化エラーハンドリング | エラーコード + メッセージの構造化パターンが存在する | 推奨 |
| S4 | ディレクトリ README | 主要ディレクトリに README.md が存在する（2つ以上） | 推奨 |
| S5 | validate スクリプト | `package.json` の scripts に `validate` 相当（型チェック+リント+テスト一括）がある | 推奨 |

### Phase 3: 検証基盤チェック（5項目）

| # | チェック項目 | 合格基準 | 重み |
|---|---|---|---|
| V1 | テストが存在する | テストファイルが5つ以上存在する | 必須 |
| V2 | Pre-commit フック | `.husky/`, `.git/hooks/`, `lefthook.yml`, `lint-staged` 等が設定されている | 推奨 |
| V3 | CI 設定がある | `.github/workflows/`, `.gitlab-ci.yml`, `Jenkinsfile` 等が存在する | 推奨 |
| V4 | カバレッジ設定 | `jest.config`, `vitest.config`, `nyc`, `c8` 等にカバレッジ設定がある | 推奨 |
| V5 | PR テンプレート | `.github/pull_request_template.md` または `.github/PULL_REQUEST_TEMPLATE/` が存在する | 推奨 |

### Phase 4: 「なぜ」チェック（追加品質）

CLAUDE.md 内のルール・規約に「なぜ」（理由）が付記されているかを確認:

| # | チェック項目 | 合格基準 |
|---|---|---|
| Q1 | ルールに理由が付いている | 主要ルールの50%以上に理由（括弧書き、「理由:」、「〜のため」等）がある |
| Q2 | 権限境界が明確 | 「自由にやってよい」「確認が必要」「禁止」の3段階が区別されている |

## Phase 5: スコア算出とレポート

### スコア計算

- 必須項目（D1, D2, D3, V1）: 各2点（合計8点）
- 推奨項目（D4, D5, S1-S5, V2-V5）: 各1点（合計11点）
- 品質項目（Q1, Q2）: 各1点（合計2点）
- **満点: 21点**

### レベル判定

| スコア | レベル | 意味 |
|---|---|---|
| 17-21 | AI-Ready | AI エージェントが高品質な出力を安定して生成できる |
| 12-16 | Almost Ready | 基本は整っている。改善で大幅な品質向上が見込める |
| 8-11 | Needs Work | AI は使えるが、文脈不足で品質にムラがある |
| 0-7 | Not Ready | AI エージェントへの文脈提供が不十分 |

### レポート出力形式

```
## CLAUDE.md 品質レポート

### スコア: {点数}/20 — {レベル}

### ドキュメント品質 ({n}/6)
| # | 項目 | 結果 | 詳細 |
|---|---|---|---|
| D1 | ファイル存在 | ✅ / ❌ | ... |
| ... |

### リポジトリ構造 ({n}/5)
| # | 項目 | 結果 | 詳細 |
|---|---|---|---|

### 検証基盤 ({n}/8)
| # | 項目 | 結果 | 詳細 |
|---|---|---|---|

### 品質 ({n}/2)
| # | 項目 | 結果 | 詳細 |
|---|---|---|---|

### 改善提案（優先度順）

1. 🔴 {必須で未達の項目}: {具体的な改善アクション}
2. 🟡 {推奨で未達の項目}: {具体的な改善アクション}
3. ...

### 次のアクション
- `/claude-md-gen update` で不足セクションを自動追加できます
```

## 注意事項

- チェックは **読み取り専用**。ファイルの変更は行わない
- 各チェック項目は Grep / Glob / Read で機械的に判定する
- 判定が曖昧な場合は「⚠️ 要確認」として手動確認を促す
- `.claude/rules/` 内のルールファイルも補助的に参照する（CLAUDE.md に統合されていない規約がないか確認）
